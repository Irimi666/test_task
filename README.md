# Тестовое задание

## Кейс : Проектирование механизма  отправки уведомлений о доставке.

### Контекст:

"Онлайн-маркетплейс товаров. Необходимо реализовать отправку пуш-уведомлений в мобильное приложение об изменении статуса заказа. Данные об изменении статуса получать из API внешнего сервиса."

### Задача:
1. Описать функциональные и нефункциональные требования к процессу, включая описание бизнес-процесса и статусную модель (диаграммы). 

2. Описать структуру API взаимодействия с внешним сервисом (опирайтесь на здравый смысл).
